### üîπ 1. **What Is Windows Authentication?**

- Uses the **Windows credentials** of users to authenticate.
- Common in **intranet environments** (corporate networks).
- Supports **Kerberos** or **NTLM** protocols.
- Eliminates the need for login forms ‚Äî users are automatically signed in using their domain credentials.

---

### üîπ 2. **Use Cases**

- Internal enterprise web apps (e.g., HR systems, dashboards).
- Apps hosted on **IIS** or **HTTP.sys** that integrate with **Active Directory**.
- When **Single Sign-On (SSO)** is required across corporate apps.

### üîπ 4. **Enable in the Project File**

For **Windows Authentication** (and disabling anonymous), edit your project‚Äôs `.csproj` file:

```xml

  <TargetFramework>net8.0</TargetFramework>
  <UseWindowsService>true</UseWindowsService>
```

### üîπ 5. **Configure Authentication in `Program.cs`**

### üîπ 7. **Use Windows Authentication in IIS**

- Open **IIS Manager** ‚Üí Select your app.
- Go to **Authentication**:
  - Enable: ‚úÖ *Windows Authentication*
  - Disable: ‚ùå *Anonymous Authentication*

Ensure **web.config** has:

```xml

  <security>
    <authentication>
      <windowsAuthentication enabled="true" />
      <anonymousAuthentication enabled="false" />
    </authentication>
  </security>
</system.webServer>
```

### üîπ 8. **Use Windows Authentication with HTTP.sys**

If you're using HTTP.sys (instead of Kestrel), configure the listener:

```csharp

{
    options.Authentication.Schemes = AuthenticationSchemes.NTLM | AuthenticationSchemes.Negotiate;
    options.Authentication.AllowAnonymous = false;
});
```

### üîπ 10. **Common Gotchas**

- **Windows Authentication** doesn‚Äôt work with **Kestrel** alone. You must use a reverse proxy (IIS, HTTP.sys).
- Don‚Äôt combine Windows Auth with other schemes unless explicitly required.
- Windows Authentication is **not suitable for public-facing apps**.
